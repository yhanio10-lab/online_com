# 현재 구현 요약

## 1. 실행

프로젝트 폴더에서 실행한다.

```powershell
cd D:\online_com\21_sku_management
npm.cmd start
```

기본 접속 주소:

```text
http://127.0.0.1:3000/admin/index.html
```

포트를 바꾸려면:

```powershell
$env:PORT=3008
npm.cmd start
```

## 2. 화면 구성

관리자 화면은 하나의 웹 화면에서 모듈을 전환하는 구조다.

```text
SKU 매핑      /admin/index.html#sku-mapping
세트/번들     /admin/index.html#sku-bundles
매출/수익     /admin/index.html#sales-profit
```

새 기능은 `public/admin/modules/manifest.json`에 모듈을 추가하는 방식으로 붙일 수 있다. 기존 화면 코드를 직접 섞지 않는 구조다.

## 3. SKU 매핑

목적은 플랫폼 상품 옵션을 이카운트 품번코드인 SKU에 연결하는 것이다.

기본 원칙:

```text
상품 옵션 1개 = SKU 1개
```

주요 기능:

- 플랫폼, 상태, 검색어 필터
- 상품별 그룹 보기
- 옵션별 보기
- 그룹 접기/펼치기
- 그룹 선택 및 옵션 선택
- SKU 직접 입력
- SKU 검색 팝업
- SKU 검색어 AND 조건 검색
- SKU 검색 결과 다중 선택
- 선택한 옵션 순서와 선택한 SKU 순서대로 1:1 반영
- 입력된 SKU 일괄 저장
- 단건 저장
- 충돌 행 강조

SKU 검색 결과에는 이카운트 품목명과 규격 정보를 함께 보여준다. 입고/출고 정보와 판매자 상품코드는 검색 결과에서 제외했다.

## 4. 세트/번들 관리

세트 SKU와 구성품 SKU를 별도로 관리한다.

주요 기능:

- 세트 SKU 검색
- 구성품 SKU 추가
- 구성 수량 수정
- 구성품 비활성 처리
- 구성 원가 계산
- 세트/번들 기준 가용재고 계산 기반

세트 SKU의 원가는 구성품 SKU의 매입단가와 구성 수량을 합산해서 계산한다.

```text
세트 원가 = 구성품1 매입단가 * 수량 + 구성품2 매입단가 * 수량 + ...
```

## 5. 엑셀 임포트

### 이카운트 품목

이카운트 품목 엑셀을 가져와 SKU 마스터를 만든다.

```powershell
npm.cmd run import:ecount -- "D:\online_com\21_sku_management\이카운트_품목.xlsx"
```

관리 항목:

- SKU 코드
- 품목명
- 규격
- 매입단가
- 판매단가
- 세트 여부
- 재고관리 여부

### 스마트스토어 상품 리스트

스마트스토어 상품 엑셀을 가져와 상품과 옵션을 만든다.

```powershell
npm.cmd run import:smartstore -- "D:\online_com\21_sku_management\스마트스토어상품_20260505_105831.xlsx"
```

플랫폼별 엑셀 양식은 다를 수 있으므로, 플랫폼별 import service를 따로 두는 방식으로 확장한다.

### 플레이오토 매출

플레이오토 매출 엑셀을 가져와 매출/수익 집계에 사용한다.

```powershell
npm.cmd run import:playauto -- "E:\내 드라이브\100.오픈마켓 매출자료\00.플레이오트\2026\2026_03\20260303\2026-03-02_EMP_롯데택배1.xls"
```

관리 화면에서도 `매출/수익` 메뉴에서 파일 경로를 입력하고 가져올 수 있다.

개인정보성 컬럼은 저장하지 않는다.

저장하지 않는 항목:

- 수령자명
- 전화번호
- 휴대폰번호
- 주소
- 배송메시지

## 6. 매출/수익 계산 기준

매출/수익 화면은 플레이오토 매출 데이터를 기준으로 집계한다.

주요 컬럼:

```text
매출액 | 매핑매출 | 미매핑매출 | 원가 | 수수료 | 수익금액 | 전체수익률 | 계산대상수익률
```

용어:

```text
매출액 = 플랫폼 전체 매출
매핑매출 = SKU 매핑이 성공한 주문 아이템의 매출
미매핑매출 = SKU 매핑이 실패한 주문 아이템의 매출
원가 = 매핑된 SKU의 매입단가 * 판매수량
수수료 = 매핑매출 * 플랫폼 수수료율
수익금액 = 매핑매출 - 원가 - 수수료
전체수익률 = 수익금액 / 전체 매출
계산대상수익률 = 수익금액 / 매핑매출
```

중요한 기준:

- SKU 매핑이 안 된 매출은 원가를 알 수 없으므로 수익 계산에서 제외한다.
- 따라서 매핑이 적으면 `전체수익률`은 낮게 나온다.
- 엑셀에서 원가가 잡힌 상품 기준 수익률을 보고 있다면 `계산대상수익률`과 비교한다.

예시:

```text
전체 매출: 2,260,820
매핑매출: 39,600
미매핑매출: 2,221,220
원가: 23,100
수수료: 2,376
수익금액: 14,124

전체수익률 = 14,124 / 2,260,820 = 0.6%
계산대상수익률 = 14,124 / 39,600 = 35.7%
```

## 7. 플랫폼 수수료

`매출/수익` 화면에서 플랫폼별 수수료율을 입력한다.

입력 단위:

```text
%
```

예:

```text
6 입력 = 6%
```

수수료율 저장 후 집계표는 즉시 다시 계산된다.

## 8. 상세내역과 다운로드

집계표의 `전체수익률` 또는 `계산대상수익률` 버튼을 클릭하면 해당 구분의 전체 상세내역이 화면 아래에 표시된다.

상세내역 항목:

```text
일자
플랫폼
주문번호
상품코드
상품명
옵션
SKU
수량
매출액
원가
수수료
수익
수익률
상태
```

`엑셀 다운로드` 버튼을 누르면 현재 표시된 상세내역을 CSV 파일로 내려받는다. Excel에서 바로 열 수 있도록 UTF-8 BOM을 포함한다.
