# 실행가이드

## 1. 서버 실행

PowerShell:

```powershell
cd D:\online_com\21_sku_management
npm.cmd start
```

포트를 지정하려면:

```powershell
cd D:\online_com\21_sku_management
$env:PORT=3008
npm.cmd start
```

## 2. 관리자 화면

```text
http://127.0.0.1:3008/admin/index.html
```

SKU 매핑:

```text
http://127.0.0.1:3008/admin/index.html#sku-mapping
```

세트/번들:

```text
http://127.0.0.1:3008/admin/index.html#sku-bundles
```

## 3. 테스트 실행

```powershell
cd D:\online_com\21_sku_management
npm.cmd test
```

## 4. 이카운트 품목 엑셀 가져오기

```powershell
cd D:\online_com\21_sku_management
npm.cmd run import:ecount -- "D:\online_com\21_sku_management\이카운트_품목.xlsx"
```

## 5. 스마트스토어 상품 엑셀 가져오기

```powershell
cd D:\online_com\21_sku_management
npm.cmd run import:smartstore -- "D:\online_com\21_sku_management\스마트스토어상품_20260505_105831.xlsx"
```

## 6. SKU 매핑 작업 순서

1. 관리자 화면 접속
2. `SKU 매핑` 메뉴 선택
3. 플랫폼 필터 선택
4. 보기 방식을 `상품별 그룹`으로 선택
5. 상품 그룹에서 `미매핑 선택` 또는 체크박스 선택
6. SKU 검색 버튼 클릭
7. SKU 검색어 입력
8. SKU 검색 결과를 옵션 순서에 맞춰 클릭
9. `선택 순서대로 적용` 클릭
10. `입력된 SKU 일괄저장` 클릭

## 7. 단일 행 저장

1. 옵션 행의 SKU 칸에 직접 입력
2. `저장` 클릭

## 8. 선택 행 저장

1. 옵션 행 체크
2. SKU 입력
3. `선택 일괄 매핑` 클릭

## 9. 입력값 전체 저장

1. SKU 검색 또는 직접 입력으로 여러 행에 SKU 입력
2. 체크 여부와 관계없이 `입력된 SKU 일괄저장` 클릭

## 10. 세트/번들 작업 순서

1. `세트/번들` 메뉴 선택
2. 세트 SKU 검색
3. 세트 SKU 선택
4. 구성품 SKU 입력
5. 구성 수량 입력
6. 추가 클릭
7. 필요 시 수량 수정 후 저장

## 11. 플레이오토 매출 가져오기

```powershell
cd D:\online_com\21_sku_management
npm.cmd run import:playauto -- "E:\내 드라이브\100.오픈마켓 매출자료\00.플레이오트\2026\2026_03\20260303\2026-03-02_EMP_롯데택배1.xls"
```

관리자 화면:

```text
http://127.0.0.1:3008/admin/index.html#sales-profit
```

작업 순서:

1. 매출/수익 메뉴 선택
2. 파일 경로 입력
3. 플레이오토 가져오기 클릭
4. 플랫폼별 또는 SKU별 집계 조회
5. 매핑 실패 목록 확인
6. SKU 매핑 화면에서 실패 옵션을 매핑
7. 같은 파일 재임포트

## 12. 데이터 저장 위치

현재 MVP는 JSON 파일을 저장소로 사용한다.

```text
data/dev.json
```

운영 DB 전환 시 SQL 기준 파일:

```text
src/db/migrations/001_sku_management.sql
```
