# 엑셀임포트명세서

## 1. 이카운트 품목 엑셀

### 목적
이카운트 품목 정보를 SKU 마스터로 가져온다.

### 실행

```powershell
npm.cmd run import:ecount -- "D:\online_com\21_sku_management\이카운트_품목.xlsx"
```

### 컬럼 매핑

| 엑셀 컬럼 | 시스템 컬럼 |
| --- | --- |
| 품목코드 | sku_code |
| 품목명 | sku_name |
| 규격 | spec |
| 단위 | unit |
| 바코드 | barcode |
| 품목구분 | item_type |
| 세트여부 | is_set |
| 재고수량관리 | inventory_managed |
| 입고단가 | purchase_price |
| 입고단가 VAT포함여부 | purchase_price_vat_included |
| 출고단가 | sale_price |
| 출고단가 VAT포함여부 | sale_price_vat_included |
| 상품위치 | location |

### 처리 규칙
- 품목코드가 없으면 스킵한다.
- 품목명이 없으면 스킵한다.
- 기존 SKU는 업데이트한다.
- 신규 SKU는 생성한다.
- 입고단가 또는 출고단가 변경 시 가격 이력을 저장한다.

## 2. 스마트스토어 상품 엑셀

### 목적
스마트스토어 상품과 옵션을 `products`, `product_options`로 가져온다.

### 실행

```powershell
npm.cmd run import:smartstore -- "D:\online_com\21_sku_management\스마트스토어상품_20260505_105831.xlsx"
```

### 파일 구조
- 2행: 실제 헤더
- 6행부터: 상품 데이터

### 컬럼 매핑

| 엑셀 컬럼 | 시스템 컬럼 |
| --- | --- |
| 상품번호 | products.product_id |
| 판매자 상품코드 | products.seller_product_code |
| 카테고리코드 | products.category_code |
| 상품명 | products.product_name |
| 상품상태 | products.status |
| 판매가 | products.sale_price |
| 재고수량 | product_options.stock_quantity |
| 옵션형태 | 옵션 생성 방식 판단 |
| 옵션명 | product_options.raw_option_name |
| 옵션번호 | product_options.option_id |
| 옵션값 | product_options.option_name |
| 옵션가 | product_options.option_price |
| 옵션 재고수량 | product_options.stock_quantity |
| 옵션 사용여부 | product_options.status |
| 부가세 | products.tax_type |
| 브랜드 | products.brand |
| 제조사 | products.manufacturer |

### 옵션 처리
옵션번호/옵션값/옵션재고가 줄바꿈으로 여러 개 있으면 각각 별도 옵션으로 펼친다.

예:

```text
옵션번호:
111
222

옵션값:
블랙
화이트
```

결과:

```text
option_id = 111, option_name = 블랙
option_id = 222, option_name = 화이트
```

### 처리 규칙
- 상품번호 또는 상품명이 없으면 스킵한다.
- 기존 상품은 업데이트한다.
- 신규 상품은 생성한다.
- 기존 옵션은 업데이트한다.
- 신규 옵션은 생성한다.

## 3. 플레이오토 매출 엑셀

### 목적
플레이오토에서 수집한 플랫폼별 매출 데이터를 가져와 수익 기초 데이터를 만든다.

### 실행

```powershell
npm.cmd run import:playauto -- "E:\내 드라이브\100.오픈마켓 매출자료\00.플레이오트\2026\2026_03\20260303\2026-03-02_EMP_롯데택배1.xls"
```

### 파일 형식
구형 `.xls` 파일은 Excel COM으로 CSV 변환 후 임포트한다.

### 사용 컬럼

| 엑셀 컬럼 | 시스템 컬럼 |
| --- | --- |
| 판매사이트명 | platform |
| 상품명 | product_name |
| 주문선택사항 | option_name |
| 추가구매옵션 | add_option_name |
| 주문수량 | quantity |
| 판매가 | gross_sales_amount |
| 주문고유번호 | order_id |
| 판매사이트 상품코드 | platform_product_id |

### 저장하지 않는 컬럼
아래 개인정보 컬럼은 저장하지 않는다.

```text
수령자명
수령자전화번호
수령자휴대폰번호
배송지우편번호
배송지주소
배송메세지
```

### 플랫폼 변환

| 판매사이트명 | platform |
| --- | --- |
| 스마트스토어 | smartstore |
| 쿠팡(신) | coupang |
| 카페24(신) | cafe24 |
| 11번가 | 11st |
| G마켓 | gmarket |
| 옥션 | auction |
| 올웨이즈 | allways |

### 매칭
1. `platform + 판매사이트 상품코드`로 상품을 찾는다.
2. 주문선택사항에서 `종류:`, `색상:` 같은 라벨을 제거한다.
3. 정규화된 옵션명으로 `product_options`를 찾는다.
4. 활성 SKU 매핑을 찾는다.
5. SKU 입고단가로 원가를 계산한다.

### 수익 계산

```text
매출액 = 판매가
원가 = SKU 입고단가 × 주문수량
수익금액 = 매출액 - 원가
수익률 = 수익금액 / 매출액
```

매핑 실패 건은 수익 계산에서 제외하고 수익금액 0으로 저장한다.
