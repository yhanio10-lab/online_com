# 기능명세서

## 1. SKU 매핑

### 목적
플랫폼 상품 옵션을 이카운트 품번코드(SKU)에 연결한다.

### 기본 규칙
- 매핑 단위는 상품 옵션이다.
- 옵션 1개는 활성 SKU 1개에 매핑한다.
- 존재하지 않는 SKU는 매핑할 수 없다.
- 비활성 SKU는 매핑할 수 없다.
- 동일 옵션에 활성 매핑이 여러 개 있으면 충돌로 표시한다.
- 매핑 생성/수정/삭제 이력은 저장한다.

### 주요 기능
- 옵션 목록 조회
- 플랫폼/상태/검색어 필터
- 상품별 그룹 보기
- 옵션별 보기
- 인라인 SKU 입력
- SKU 검색 팝업
- SKU 검색 AND 조건
- SKU 검색 결과 순서 선택
- 선택된 옵션 순서와 SKU 선택 순서대로 1:1 적용
- 체크된 옵션 일괄 매핑
- 입력된 SKU 일괄저장
- 충돌 행 강조

## 2. 상품별 그룹 보기

### 그룹 기준
`platform + platform_product_id`

### 그룹 행 표시
- 플랫폼
- 플랫폼 상품번호
- 상품명
- 옵션 수
- 매핑 수
- 미매핑 수
- 충돌 수

### 동작
- 그룹 체크박스 선택 시 하위 옵션 전체 선택
- 하위 옵션 일부 선택 시 그룹 체크박스는 부분 선택 상태
- 그룹 접기/펼치기 가능
- 미매핑 선택 버튼으로 해당 상품의 미매핑 옵션만 선택

## 3. SKU 검색

### 검색 조건
공백으로 나눈 키워드를 모두 만족해야 한다.

예:

```text
프릭션 0.5 블랙
```

검색 대상:

```text
sku_code
sku_name
spec
barcode
```

### 검색 결과 표시
- SKU 코드
- 품목명
- 이카운트 규격
- 세트 SKU인 경우 세트 표시

입고단가/출고단가는 SKU 검색 결과에 표시하지 않는다.

### 다중 적용
옵션 행을 여러 개 선택한 상태에서 SKU 검색 결과를 여러 개 클릭하면 선택 순번이 표시된다.

```text
옵션 선택 순서 1 -> SKU 선택 순서 1
옵션 선택 순서 2 -> SKU 선택 순서 2
옵션 선택 순서 3 -> SKU 선택 순서 3
```

옵션 수와 SKU 선택 수가 다르면 적용하지 않는다.

## 4. 일괄 저장

### 선택 일괄 매핑
체크된 옵션 행만 저장한다.

### 입력된 SKU 일괄저장
체크 여부와 무관하게 현재 화면에서 SKU 값이 입력되었거나 변경된 행을 저장한다.

## 5. 세트/번들 관리

### 목적
세트 SKU의 구성품 SKU와 구성 수량을 관리한다.

### 기본 규칙
- 세트 SKU는 `sku_master.is_set = true`로 구분한다.
- 구성품은 `sku_bundle_components`에 저장한다.
- 세트 SKU는 자기 자신을 구성품으로 가질 수 없다.
- 구성 수량은 1 이상이어야 한다.

### 화면 기능
- 세트 SKU 목록
- 구성 완료/미완료 KPI
- 구성품 추가
- 구성 수량 수정
- 구성품 비활성 처리
- 구성원가/마진 표시

## 6. 매출 집계 기초

### 목적
주문 아이템을 옵션 기준으로 SKU에 정규화하고 SKU 기준 매출 집계를 만든다.

### 처리 규칙
- 주문 아이템의 `platform + option_id`로 옵션을 찾는다.
- 옵션의 활성 SKU 매핑을 찾는다.
- 매핑 실패 아이템은 `mapping_failed`로 분리한다.

## 7. 재고 집계 기초

### 목적
SKU 기준 현재 가용재고를 계산한다.

### 계산

```text
가용재고 = 최근 스냅샷 현재고 + 이후 이동수량 합계 - 예약재고
```

### 세트 가용재고

```text
세트 가용재고 = 구성품별 가용재고 / 구성수량 중 최솟값
```
